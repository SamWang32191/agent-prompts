# 工作流程模式

## 順序工作流程

對於複雜的任務，將操作分解為清晰的順序步驟。在 SKILL.md 的開頭提供流程的概覽給 Claude 通常會很有幫助：

```markdown
填寫 PDF 表單包含以下步驟：

1. 分析表單（執行 analyze_form.py）
2. 建立欄位對應（編輯 fields.json）
3. 驗證對應（執行 validate_fields.py）
4. 填寫表單（執行 fill_form.py）
5. 驗證輸出（執行 verify_output.py）
```

## 條件工作流程

對於具有分支邏輯的任務，引導 Claude 通過決策點：

```markdown
1. 確定修改類型：
   **建立新內容？** → 遵循下方的「建立工作流程」
   **編輯現有內容？** → 遵循下方的「編輯工作流程」

2. 建立工作流程：[步驟]
3. 編輯工作流程：[步驟]
```
